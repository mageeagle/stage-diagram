# Implementation Plan: Location-based Sub-flows

## Goal
Create a feature that groups nodes with the same `Location` into sub-flows (groups) with a label. This can be toggled on and off.

## Steps

### 1. Update Store (`src/store/useStore.ts`)
- Add `locationGroupsEnabled: boolean` to the `Store` state.
- Add `toggleLocationGroups: () => void` to the `Store` actions.

### 2. Create a Group Node Component (`src/components/nodes/GroupNode.tsx`)
- Create a new custom node component to represent the sub-flow container.
- The node should:
    - Have a background color/style.
    - Display a label (the location name) at the top.
    - Be non-interactive/non-draggable in a way that doesn't interfere with children (or just standard React Flow behavior).

### 3. Implement Grouping Logic in `DiagramCanvas.tsx`
- Use `useMemo` to derive `displayNodes` and `displayEdges` from the state.
- **If `locationGroupsEnabled` is `true`**:
    - Group nodes by their `data.location`.
    - For each group:
        - Calculate the bounding box (min X, min Y, max X, max Y) of all nodes in the group.
        - Create a `GroupNode` with the calculated dimensions and position.
        - For each node in the group, set its `parentId` to the group node's ID.
        - **Important**: Convert the node's `position` from absolute to relative to the group node's position.
    - Add the `GroupNode`s to the `displayNodes`.
- **If `locationGroupsEnabled` is `false`**:
    - `displayNodes` are just the original `nodes`.
- **Handle node changes**:
    - Override `onNodesChange` to intercept changes to "relative" positions and convert them back to "absolute" positions in the source `nodes` state.
    - This ensures that when the user drags a node inside a group, the underlying `nodes` state is updated with its absolute position.

### 4. Add Toggle UI
- Add a toggle switch in the `Panel` within `DiagramCanvas.tsx` to control `locationGroupsEnabled`.

### 5. Verification
- Test grouping with multiple nodes having the same location.
- Test toggling the feature on and off.
- Verify that node positions are preserved correctly through toggling.
- Verify that labels are correctly displayed.

## Dependencies
- `reactflow` for grouping/parent-child functionality.
- `src/store/useStore.ts` for state management.
