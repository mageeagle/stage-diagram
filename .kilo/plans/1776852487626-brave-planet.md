# Implement Save and Restore Functionality

This plan outlines the steps to implement a feature that allows users to save and restore their entire project state, including templates and the current canvas (nodes and edges).

## Goal
Implement a "Save Project" (export to JSON) and "Load Project" (import from JSON) functionality.

## Proposed Data Structure (Project JSON)
The exported file will follow this schema:
```json
{
  "version": "1.0",
  "templates": [
    {
      "id": "string",
      "name": "string",
      "nodes": "Node[]",
      "edges": "Edge[]"
    }
  ],
  "nodes": "Node[]",
  "edges": "Edge[]"
}
```

## Implementation Steps

### 1. Create Project I/O Utilities
Create `src/utils/projectIO.ts` to encapsulate the logic for exporting and importing the project state.
- `exportProject(state: { templates: Template[], nodes: Node[], edges: Edge[] }): void`
    - Converts the state to a JSON string.
    - Creates a Blob and triggers a browser download for `project.json`.
- `importProject(file: File): Promise<ProjectState>`
    - Reads the file as text.
    - Parses JSON.
    - Validates the structure (presence of `templates`, `nodes`, and `edges`).
    - Returns the parsed state.

### 2. Update Store Actions (if needed)
Check `src/store/useStore.ts` to ensure we can atomically update the nodes, edges, and templates during a restore.
- The store already has `setNodesAndEdges` and `setTemplates`. I will use these.

### 3. Implement UI in SettingsModal
Add a new "Project" section to `src/components/settings/SettingsModal.tsx`.
- **Save Project Button**: Calls `exportProject` with the current state from `useStore`.
- **Load Project Button**: 
    - Triggers a hidden `<input type="file" accept=".json">`.
    - On file change, calls `importProject`.
    - If successful, calls `setNodesAndEdges` and `setTemplates` in the store.
    - Handle errors (invalid JSON, wrong format) with a simple alert or error message.

### 4. Verification
- Manually test the save/load cycle.
- Verify that templates are correctly restored.
- Verify that node positions and connections are correctly restored.
- Verify that the canvas is in the exact same state after loading.
