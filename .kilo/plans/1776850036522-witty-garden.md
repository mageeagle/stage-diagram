# Undo/Redo System Plan

This plan outlines the implementation of an undo/redo system for adding and deleting nodes (and edges) in the node diagram application.

## Objectives

- Implement a history management system within the `useStore` Zustand store.
- Provide `undo` and `redo` capabilities for diagram modifications.
- Integrate keyboard shortcuts (`Ctrl+Z`, `Ctrl+Y`) in the `DiagramCanvas` component.

## Implementation Details

### 1. Store Modification (`src/store/useStore.ts`)

#### Data Structures
- Define `HistoryState` interface:
  ```typescript
  interface HistoryState {
    nodes: Node<CustomNodeData>[];
    edges: Edge[];
  }
  ```
- Add `undoStack: HistoryState[]` and `redoStack: HistoryState[]` to `DiagramState`.
- Add `undo: () => void` and `redo: () => void` to `DiagramState`.

#### Logic
- **`recordHistory`**: A helper function to be called before any state-changing action that should be undoable.
  - It will take the current `nodes` and `edges` and push them to `undoStack`.
  - It will clear `redoStack` to ensure a linear history.
  - It will cap the `undoStack` size (e.g., 50 entries) to prevent excessive memory usage.

- **`undo`**:
  - If `undoStack` is not empty:
    - Pop the last `HistoryState` from `undoStack`.
    - Push the current `nodes` and `edges` to `redoStack`.
    - Set the store's `nodes` and `edges` to the popped state.

- **`redo`**:
  - If `redoStack` is not empty:
    - Pop the last `HistoryState` from `redoStack`.
    - Push the current `nodes` and `edges` to `undoStack`.
    - Set the store's `nodes` and `edges` to the popped state.

#### Affected Actions
The following actions will be wrapped with `recordHistory()` before updating the state:
- `addNode`
- `applyTemplate`
- `copyNode`
- `deleteNode`
- `deleteEdge`
- `onConnect`

### 2. Component Modification (`src/components/diagram/DiagramCanvas.tsx`)

#### Keyboard Shortcuts
- Update the `handleKeyDown` function inside the `useEffect` hook to listen for:
  - `(event.ctrlKey || event.metaKey) && event.key === 'z'`: Trigger `undo()`.
  - `(event.ctrlKey || event.metaKey) && event.key === 'y'`: Trigger `redo()`.
  - `(event.ctrlKey || event.metaKey) && event.shiftKey && event.key === 'Z'`: Trigger `redo()` (common alternative).

## Verification Plan

1. **Node Addition**: 
   - Add a node via `NodeCreationModal`.
   - Press `Ctrl + Z`. Node should be removed.
   - Press `Ctrl + Y`. Node should be restored.
2. **Node Deletion**:
   - Select a node and press `Delete`.
   - Press `Ctrl + Z`. Node should be restored.
   - Press `Ctrl + Y`. Node should be deleted again.
3. **Edge Operations**:
   - Connect two nodes.
   - Delete an edge.
   - Press `Ctrl + Z` to restore the edge.
4. **History Limit**:
   - Perform more than 50 operations and verify the application remains stable.
