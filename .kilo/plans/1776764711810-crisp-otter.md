# Theme Implementation Plan

## Goal

Implement a comprehensive light/dark mode switcher for the node diagram application, ensuring all ReactFlow elements (nodes, edges, handles, controls, and background) react correctly to theme changes.

## Investigation Results

- **Theme Management**: `useThemeStore.ts` correctly toggles the `dark` class on `document.documentElement`.
- **Custom Nodes**: `CustomNode.tsx` uses Tailwind `dark:` classes for the node container, but the `Handle` components have hardcoded colors (`!bg-blue-500`, `!bg-green-500`).
- **Edges**: Edges are created using default `addEdge`, and no explicit styling is provided in `DiagramCanvas.tsx`, making them potentially difficult to see in dark mode.
- **Controls**: The `<Controls />` component is used without any theme-specific configuration.
- **Background**: The `Background` component uses a ternary operator in `DiagramCanvas.tsx` to switch between `#333` and `#aaa`, which works but could be more integrated.
- **CSS Variables**: `app/globals.css` defines basic theme variables but doesn't include variables for ReactFlow components.

## Proposed Solution

Use a combination of Tailwind CSS dark mode classes for components and global CSS overrides for ReactFlow's internal elements.

### 1. Global Theming (CSS)
Modify `app/globals.css` to:
- Define CSS variables for ReactFlow elements (edges, controls) in both `:root` and `.dark`.
- Add global CSS rules to override ReactFlow's default styles using these variables (e.g., `.react-flow__edge-path`, `.react-flow__controls`).

### 2. Component Updates
- **`CustomNode.tsx`**: Update `Handle` components to use Tailwind's dark mode variants (e.g., `!bg-blue-500 dark:!hover:bg-blue-300`) instead of hardcoded colors.
- **`DiagramCanvas.tsx`**: 
    - Refine the `Background` color logic (ensure it's consistent).
    - (Optional) If needed, apply classes to `Controls` or `Edges`.

## Implementation Steps

1.  **Update `app/globals.css`**:
    - Add CSS variables for ReactFlow elements.
    - Add global CSS overrides for `.react-flow__edge-path` and `.react-flow__controls`.
2.  **Update `src/components/nodes/CustomNode.tsx`**:
    - Update `Handle` classes to include `dark:` variants.
3.  **Verify**:
    - Check the visual consistency of nodes, edges, handles, controls, and background in both light and dark modes.
