# Hidden Edges Feature Implementation Plan

**Date:** 2026-04-26  
**Status:** Ready for implementation  
**Priority:** High

---

## Overview

This plan implements a hidden edges feature for the node-diagram application. Edges will be hidden based on their connected nodes' hidden state, ensuring that connections involving hidden nodes are properly excluded from exports and visual representations.

**Visibility Approach:** Use CSS class `.hidden-edges { display: none }` on the edge element. This hides both the edge path and the LabeledEdge overlay because React Flow renders them in the same edge context. No transitions (as requested).

### 3. Apply Hidden Property Logic to Edges (`src/store/useStore.ts`)

**File:** `src/store/useStore.ts`  
**Location:** Line 306-318 (`updateNodeHidden` action)

**Purpose:** When a node's hidden property changes, automatically update all connected edges to reflect the new hidden state.

**Current Code:**
```typescript
updateNodeHidden: (nodeIds, hidden) => {
  set({
    nodes: get().nodes.map((node) => {
      if (nodeIds.includes(node.id)) {
        return {
          ...node,
          data: { ...node.data, hidden },
        };
      }
      return node;
    }),
  });
},
```

**Updated Code:**
```typescript
updateNodeHidden: (nodeIds, hidden) => {
  const updatedNodes = get().nodes.map((node) => {
    if (nodeIds.includes(node.id)) {
      return {
        ...node,
        data: { ...node.data, hidden },
      };
    }
    return node;
  });

  // Create a map of node IDs to their hidden state for quick lookup
  const nodeHiddenMap = new Map(
    get().nodes.map((node) => [node.id, node.data?.hidden ?? false])
  );

  const updatedEdges = get().edges.map((edge) => {
    const sourceHidden = nodeHiddenMap.get(edge.source) ?? false;
    const targetHidden = nodeHiddenMap.get(edge.target) ?? false;
    const shouldHide = sourceHidden || targetHidden;
    
    // Only update if the desired state is different
    const currentHidden = edge.data?.hidden ?? false;
    const newHidden = currentHidden === shouldHide ? currentHidden : shouldHide;
    
    return {
      ...edge,
      data: {
        ...edge.data,
        hidden: newHidden,
      },
    };
  });

  set({
    nodes: updatedNodes,
    edges: updatedEdges,
  });
},
```

**Rationale:** 
- When a node becomes hidden, ALL its connected edges become hidden
- When a node becomes visible, edges are only re-visible if BOTH connected nodes are visible
- This ensures edges are always derived from node states
- The edge's own hidden property becomes a snapshot that gets updated

**Note:** Using inline `style={{ display: "none" }}` is simpler than CSS classes for this use case, since:
- No animation issues (as user requested)
- Direct React property binding
- No CSS file maintenance

**Updated Code:**
```typescript
// Cables section
const cablesReport: ReportRow[] = [];

// Filter edges: only include if edge and both nodes are not hidden
const filteredEdges = edges.filter((edge) => {
  const edgeHidden = edge.data?.hidden ?? false;
  
  // Check if source node is hidden
  let sourceHidden = false;
  if (edge.source) {
    const sourceNode = nodes.find((n) => n.id === edge.source);
    if (sourceNode) {
      sourceHidden = sourceNode.data?.hidden ?? false;
    }
  }
  
  // Check if target node is hidden
  let targetHidden = false;
  if (edge.target) {
    const targetNode = nodes.find((n) => n.id === edge.target);
    if (targetNode) {
      targetHidden = targetNode.data?.hidden ?? false;
    }
  }
  
  // Edge is hidden if: edge is hidden OR source node is hidden OR target node is hidden
  const isVisible = !edgeHidden && !sourceHidden && !targetHidden;
  return isVisible;
});

// Get unique cable types from filtered edges
const cableTypeCounts = new Map<string, number>();
filteredEdges.forEach((edge) => {
  const type = edge.data?.cableType as string;
  if (type && type !== 'none') {
    cableTypeCounts.set(type, (cableTypeCounts.get(type) || 0) + 1);
  }

if (cableTypeCounts.size > 0) {

    type: 'header',
    label: 'Cables',
```

### 6. LabeledEdge Considerations (Updated)

**Answer to your questions:**

1. **Do LabeledEdge labels need separate hiding?**
   - **No** - the LabeledEdge overlay will automatically hide when the edge container is hidden using `display: none`

2. **How to hide edges?**
   - **Answer:** Use `style={{ display: "none" }}` on the edge's style prop when `edge.data.hidden === true`
   - React Flow passes the style to the edge's root SVG element, which hides both the edge path and any children (including EdgeLabel)

**Why not CSS class instead?**
- CSS class works too, but inline style is simpler for this use case
- No animation/transition issues (as requested)
- Direct React state binding

**Why not return null?**
- Would hide from canvas entirely (including hit detection)
- CSS `display: none` is cleaner and preserves edge structure

**Recommendation:** 
- Use `style={{ display: "none" }}` in `DiagramCanvas.tsx` when rendering edges
- Check `edge.data.hidden` flag
- No changes needed to `LabeledEdge.tsx

## Edge Cases & Edge Cases

### Case 1: Edge with one hidden node
**Scenario:** Node A is visible, Node B is hidden, Edge A-B exists  
**Expected:** Edge is hidden  
**Implementation:** `targetHidden = true` → `shouldHide = true` → edge hidden ✓

### Case 2: Edge with two hidden nodes
**Scenario:** Both Node A and Node B are hidden, Edge A-B exists  
**Expected:** Edge is hidden  
**Implementation:** `sourceHidden = true || targetHidden = true` → `shouldHide = true` → edge hidden ✓

### Case 3: Edge deletion removes node
**Scenario:** Delete Node B, Edge A-B exists  
**Expected:** Edge is deleted (handled by deleteNodes logic)  
**Implementation:** `deleteNodes` already filters edges by source/target ✓

### Case 4: Node copy
**Scenario:** Copy a hidden node, edges are copied  
**Expected:** Copied edges reflect the new nodes' hidden state  
**Implementation:** `copyNodes` creates new edges with new IDs. Edges should be re-evaluated by `updateNodeHidden` logic if needed. May need to:
- Call `applyHiddenStateToEdges` after copy completes
- Or ensure new nodes are initialized with `hidden: false`, so edges stay visible until parent node is hidden

### Case 5: Node paste
**Scenario:** Paste a node, edges are created  
**Expected:** Pastes edges are visible unless the pasted node itself is hidden  
**Implementation:** `onConnect` is called for each pasted connection. New edges start with `hidden: false`. If parent node is hidden, subsequent `updateNodeHidden` call will hide edges.

### Case 6: Export with hidden nodes
**Scenario:** Export with some nodes hidden  
**Expected:** Only edges between visible nodes appear in export  
**Implementation:** `generateNodeListReport` filters edges based on nodes' hidden state ✓
