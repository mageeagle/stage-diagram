# Plan: Improve Group Copying Behavior

This plan addresses the issues when copying a group of nodes:
1. Connections (edges) are not copied.
2. The original nodes remain selected, causing them to move with the copy.

## Implementation Steps

### 1. Update `copyNodes` in `src/store/useStore.ts`

Modify the `copyNodes` function to:
- **Support Group IDs**: If any of the provided `nodeIds` are group IDs (starting with `group-`), identify all nodes belonging to that group (based on their `location`) and include them in the copy operation.
- **Copy Edges**: 
    - Iterate through all existing edges.
    - If an edge connects two nodes that are being copied, create a new edge between their corresponding new nodes.
    - If an edge connects one node being copied to a node outside the group, create a new edge between the new node and the original external node.
    - This ensures that the connections (both internal to the group and to external nodes) are preserved in the copy.
- **Create a New Group**: To ensure the copied nodes form a new, separate group instead of being merged into the existing one, append `" (copy)"` to the `location` property of the newly created nodes.
- **Update Selection**: After creating the new nodes and edges, update the `selectedNodeIds` state to contain only the IDs of the newly created nodes, ensuring the original nodes are deselected.

## Verification Plan

1. **Manual Verification**:
   - Create a group of nodes.
   - Connect some nodes within the group to each other.
   - Connect one node in the group to a node outside the group.
   - Select the group and press 'C' (or use the copy button).
   - **Verify**:
     - The new nodes are created with an offset position.
     - The new nodes have connections between them (internal edges).
     - The new nodes have a connection to the original external node (external edge).
     - The new nodes form a new, separate group (indicated by a new group box).
     - Only the new nodes are selected.
     - The original nodes are deselected.
2. **Regression Testing**:
   - Ensure single node copying still works as expected.
   - Ensure deleting nodes still works as expected.
