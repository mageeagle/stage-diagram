# Plan: On-Hover Tooltips for Toolbar Buttons

## Approach

## Phase 1: Create Tooltip Component

Create a new file `src/components/tooltip/Tooltip.tsx`:

- Accept `position`, `isVisible`, `content`, `className` as props
- Render an absolute positioned popup that follows mouse movement
- Use React hooks (`useState`, `useEffect`, `useRef`) to track mouse position
- Add animation/transition for hover states
- Style with matching theme (dark/light mode support via dark: variants)
- Add slight delay on leave to allow tooltip to fully display on hover then hide

## Phase 2: Modify Toolbar

Edit `src/components/toolbar/Toolbar.tsx`:

1. Import the new `Tooltip` component
2. Create a helper array/object mapping toolbar buttons to their actions:
   - `key`: unique identifier
   - `title`: human-readable label
   - `onClick`: button click handler
   - `tooltip`: any additional description (optional)
3. Map over this array to render:
   - `<button>` element with title attribute for accessibility
   - `<Tooltip>` component with key as `updateId`, content from `title`
4. Position Tooltip absolutely relative to button container
5. Show tooltip based on mouse hover state over button

Note: We'll extract the tooltip logic into a separate component to keep Toolbar clean and maintainable.

## Phase 3: Styling

Add CSS for the tooltip:

- Initial state: `opacity: 0; transform: translateY(10px)`
- Hover state: `opacity: 1; transform: translateY(0)`
- Transition: `opacity 150ms, transform 150ms`
- Background color: `bg-stone-50 dark:bg-stone-700`
- Border and shadow similar to button styling
- Padding: `px-2 py-1`
- Font size: `text-xs`
- Position: absolute, rounded-md, shadow-sm
- Max-width for longer titles

## Phase 4: Accessibility Considerations

- Maintain existing `title` attributes for screen readers
- Tooltips are supplementary information, not replacements
- Add aria-describedby linking tooltip content when visible
- Ensure sufficient contrast for visibility

## Deliverables

1. New `src/components/tooltip/Tooltip.tsx` component
2. Modified `src/components/toolbar/Toolbar.tsx`
3. CSS styling integrated via Tailwind classes
