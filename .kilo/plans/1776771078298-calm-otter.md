# Plan: Add node copying functionality

This plan outlines the steps to implement node copying via a button and a keyboard shortcut ('C').

## 1. Update the store (`src/store/useStore.ts`)
- **Modify `DiagramState` interface**: Add `copyNode: (nodeId: string) => void;`.
- **Implement `copyNode` in `useStore`**:
  - Find the existing node by `nodeId`.
  - If found, create a new node object:
    - Use `nanoid()` for a new `id`.
    - Spread the existing node's properties (to preserve `type` and `data`).
    - Offset the `position` by `(x: 20, y: 20)` to visually distinguish it from the original.
  - Update the `nodes` array in the state by appending the new node.

## 2. Add keyboard shortcut (`src/components/diagram/DiagramCanvas.tsx`)
- **Update `handleKeyDown` in `useEffect`**:
  - Add a condition to check if `event.key.toLowerCase() === 'c'`.
  - Ensure that it only triggers if `selectedNodeId` is not null.
  - Call `copyNode(selectedNodeId)` when the condition is met.
  - Ensure that the implementation respects the `isTyping` check already present to avoid triggering while typing in inputs.

## 3. Add UI button (`src/components/inspector/PropertyInspector.tsx`)
- **Import `Copy` icon**: Add `Copy` to the `lucide-react` imports.
- **Destructure `copyNode`**: Retrieve `copyNode` from `useStore`.
- **Add "Copy Node" button**:
  - Place a new button in the bottom action area of the inspector.
  - Style it with blue colors (`text-blue-600 bg-blue-50 hover:bg-blue-100`) to distinguish it from the red "Delete Node" button.
  - Use the `Copy` icon and label it "Copy Node".
  - Wrap the bottom buttons in a container with `space-y-2` to manage spacing.
