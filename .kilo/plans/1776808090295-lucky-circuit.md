# Plan: Add functionality to delete selected connection with Delete/Backspace key

## Overview
The goal is to allow users to delete a selected connection (edge) in the ReactFlow diagram by pressing the `Delete` or `Backspace` key, similar to how nodes can currently be deleted.

## Implementation Steps

### 1. Update `src/store/useStore.ts`
- Modify `DiagramState` interface to include:
    - `selectedEdgeId: string | null`
    - `setSelectedEdge: (edgeId: string | null) => void`
    - `deleteEdge: (edgeId: string) => void`
- Implement `setSelectedEdge` and `deleteEdge` in `useStore`.
    - `setSelectedEdge` will set the `selectedEdgeId` in the state.
    - `deleteEdge` will filter out the edge with the given ID from the `edges` array.

### 2. Update `src/components/diagram/DiagramCanvas.tsx`
- Destructure `selectedEdgeId`, `setSelectedEdge`, and `deleteEdge` from `useStore`.
- Implement an `onSelectionChange` callback for the `ReactFlow` component:
    - Use the `Selection` object provided by ReactFlow to identify the selected edge.
    - Call `setSelectedEdge` with the ID of the selected edge, or `null` if no edge is selected.
- Pass `onSelectionChange` to the `ReactFlow` component.
- Update the `handleKeyDown` function inside `useEffect`:
    - Check if `event.key` is `"Delete"` or `"Backspace"`.
    - If `selectedEdgeId` is present, call `deleteEdge(selectedEdgeId)`.
    - To avoid conflicts with node deletion, I will decide on the priority. If both a node and an edge are selected, deleting the node also deletes its connected edges. I will check if `selectedEdgeId` is present first, or if I should attempt to delete both. 
    - Actually, the current implementation for node deletion is:
      ```tsx
      if ((event.key === "Delete" || event.key === "Backspace") && selectedNodeId) {
        deleteNode(selectedNodeId);
      }
      ```
      I will add:
      ```tsx
      if ((event.key === "Delete" || event.key === "Backspace") && selectedEdgeId) {
        deleteEdge(selectedEdgeId);
      }
      ```
      If both are selected, the current logic will delete the node first because it's checked first. This is actually fine because deleting the node also deletes its edges. However, if the user *only* wants to delete the edge when it's selected, I might want to check `selectedEdgeId` first. But if both are selected, deleting the node is more "complete". I'll stick to checking both.

## Verification
- Manually test in the application:
    - Click an edge to select it.
    - Press `Delete` or `Backspace`.
    - Verify the edge is removed from the diagram.
    - Verify that selecting a node still allows deleting the node.
    - Verify that selecting both a node and an edge and pressing delete works as expected (deleting the node should be fine).
