# Plan: Add Power Summary Rows to NodeListModal

This plan implements the requirement to add power-related summary rows to the `NodeListModal` based on the grouping mode.

## Objectives
1. Add "Power Sockets Needed" rows when grouping by location.
2. Add "Power Breakout (Minimum)" row when not grouping by location.

## Implementation Steps

### 1. Update Data Structures
- Modify `GroupedNode` interface in `src/components/diagram/node-list-modal-types.ts` to include a `hasPower: boolean` property. This allows the summary logic to know if a group of nodes (which share name/type/location) contains nodes that require power.

### 2. Update Grouping Logic
- Update `groupByName`, `groupByLocation`, and `groupByType` in `src/components/diagram/node-list-utils.ts` to populate the `hasPower` property in the `GroupedNode` objects.
  - A `GroupedNode` will have `hasPower: true` if any of the nodes being merged into that group have `data.power === true`.

### 3. Update `NodeListContent.tsx`
Refactor the rendering logic in `NodeListContent.tsx` to handle the summary rows.

#### Case A: `groupBy === 'location'`
- Iterate through `aggregatedNodes` to build the list of components.
- Maintain a running count of `powerSocketsInCurrentLoc` (sum of `quantity` where `hasPower` is true) for the current location.
- When the location changes (or at the end of the list), insert a summary row:
  - **Label**: "Power Sockets Needed"
  - **Value**: The calculated quantity.
  - **Styling**: A distinct summary row style (e.g., bold, slightly different background) to distinguish it from regular items.

#### Case B: `groupBy !== 'location'`
- Render the nodes and headers (for `type`) normally.
- At the end of the list, calculate the total number of unique locations that have at least one node with `hasPower: true`.
- Insert a final summary row:
  - **Label**: "Power Breakout (Minimum)"
  - **Value**: The count of unique locations.
  - **Styling**: A distinct summary row style.

## Verification
- Verify that "Power Sockets Needed" appears correctly at the end of each location group when `groupBy === 'location'`.
- Verify that "Power Breakout (Minimum)" appears once at the end of the list when `groupBy !== 'location'`.
- Ensure the quantities calculated match the expected values based on the `power` property of the nodes.
- Ensure styling is consistent with the existing UI.
