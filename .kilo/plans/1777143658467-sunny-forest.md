# Plan: Multi-Edge Selection UI Enhancement

## Current Problems (Confirmed by User)

1. **Multi-select editing only works for nodes**, NOT edges
2. **Dropdowns only show when ONLY multiple edges are selected** (no nodes selected)
3. **Need to display dropdowns when BOTH multiple nodes and multiple edges are selected**

## Desired Behavior

EdgePropertiesPanel should ALWAYS show EdgeType and CableType dropdowns when ANY edges are selected (1+), regardless of whether nodes are also selected. When multiple edges are selected, use common values and apply to all edges simultaneously.

## Implementation Plan

### Changes to EdgePropertiesPanel (`src/components/inspector/EdgePropertiesPanel.tsx`):

1. **Remove condition** that renders panel only when no nodes selected (line 21: `if (selectedEdges.length === 0) return null;` should always be checked, but panel should render when `selectedEdges.length > 0`)

2. **Keep existing multi-select logic** - the current `getCommonCableType()` and `getCommonEdgeType()` functions already handle:
   - Single select: show individual edge's type
   - Multi-select with same type: show that common type
   - Multi-select with different types: show "mixed"/default with warning

3. **No changes to store** - `updateEdgeType` and `updateEdgeCableType` already accept edgeId arrays and update all matching edges

### Summary:
- Edit EdgePropertiesPanel to always render when edges are selected (not just when no nodes selected)
- Existing logic handles single vs multi-select correctly
- Changes apply to all selected edges at once
