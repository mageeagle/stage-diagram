# Plan: Fix Theme Reactivity for Custom Nodes and Background

## Goal
Ensure that custom nodes and the canvas background react correctly to theme changes (light/dark mode).

## Analysis
- **Custom Nodes**: Currently use Tailwind's `dark:` variant. We will switch to using the `theme` from `useThemeStore` directly with a `cn` utility to ensure reliable reactivity.
- **Background**: The user does *not* want the visual pattern from the `Background` component (keep it commented out). However, the base background color of the canvas (the area behind the nodes) is not changing. This area should ideally reflect the theme's background color.
- **ReactFlow Theme**: `ReactFlow` is receiving `colorMode={theme}`, which handles internal components like `Controls`.
- **Utility**: We will use a `cn` utility (combining `clsx` and `tailwind-merge`) to handle conditional Tailwind classes cleanly.

## Implementation Steps

1.  **Create `src/lib/utils.ts`** with the `cn` utility.
2.  **Investigate Background Color**: Determine why the background (the area behind nodes) isn't changing. If `Background` is commented out, the canvas is transparent and should show the `body` background. If `body` background isn't changing, I'll check if the `.dark` class is correctly applied to `html` and if `globals.css` is properly responding.
3.  **Update `src/components/nodes/CustomNode.tsx`** to use the `cn` utility and the `theme` from `useThemeStore` for its styling.
    - Replace `dark:bg-stone-800` with conditional classes based on `theme === 'dark'`.
    - Replace `dark:text-stone-100` with conditional classes.
    - Replace `dark:border-stone-600` with conditional classes.
    - Update the `Handle` component colors similarly.
4.  **Verify `app/globals.css`** to ensure the CSS variables for ReactFlow (like `--react-flow-edge`) and the general background are correctly defined for both modes.

## Verification
- Toggle the theme using `ThemeSwitcher`.
- Verify that `CustomNode` changes its background, text color, and border color.
- Verify that the canvas background color (the area behind the nodes) changes.
- Verify that the `Controls` component also reacts to the theme.
