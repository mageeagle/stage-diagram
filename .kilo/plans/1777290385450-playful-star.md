# Custom Shapes for StagePlan Canvas - Implementation Plan

## Architecture Clarification

StagePlan is a **completely separate view** from the main diagram:
- **DiagramCanvas** ↔ **useStore** ↔ **CustomNode** (main diagram)
- **StagePlanCanvas** ↔ **useStagePlanStore** ↔ **StagePlanCustomNode** (stage plan)

`StagePlanCanvas` renders instead of `DiagramCanvas` when `isStagePlanEnabled` is true (mutually exclusive in `app/page.tsx`).

`StagePlanCustomNode` already exists (`src/components/nodes/StagePlanCustomNode.tsx`) and already renders shapes (rectangle/circle/triangle) and rotation.

## Problem

Shape and rotation logic is currently in `useStore` and the main `PropertiesPanel`. The user wants:
1. Shape/rotation actions in **`useStagePlanStore`**
2. A **separate StagePlan PropertiesPanel** component (not the main one)
3. No coupling to `DiagramCanvas`

## Implementation Steps

### Step 1: Update `useStagePlanStore` actions and `matchNode`

**File:** `src/store/useStagePlanStore.ts`

Add to the interface:
```ts
updateNodeShape: (nodeIds: string[], shape: 'rectangle' | 'circle' | 'triangle') => void;
updateNodeRotation: (nodeIds: string[], rotation: number) => void;
updateNodeDimensions: (nodeIds: string[], width: number, height: number) => void;
```

Implement as store actions (follow `updateNodeHidden` pattern — record history, then map nodes).

**Update `matchNode`:**
Modify `matchNode` to ensure that when a node is matched from the main canvas, its StagePlan-specific properties (`shape`, `rotation`, `width`, `height`) and other essential data (`inputs`, `outputs`, `type`, `power`) are preserved in the `newNode.data` object.


### Step 2: Create `StagePlanPropertiesPanel` component

**New File:** `src/components/inspector/StagePlanPropertiesPanel.tsx`

A self-contained panel that:
- Subscribes to `useStagePlanStore` for `selectedNodeIds` and `nodes`
- Finds primary selected node from StagePlan store's nodes
- Renders **only** StagePlan-relevant controls:
  - **Shape** dropdown (rectangle/circle/triangle)
  - **Rotation** slider (0-360°)
- Calls `useStagePlanStore` actions for updates
- Does NOT show main diagram properties (type, location, inputs, outputs, power, templates)

### Step 3: Update `PropertyInspector` to conditionally render StagePlan panel

**File:** `src/components/inspector/PropertyInspector.tsx`

- Check `isStagePlanEnabled` from `useStagePlanStore`
- When StagePlan is enabled:
  - Check `useStagePlanStore.selectedNodeIds` (not `useStore.selectedNodeIds`)
  - Render `StagePlanPropertiesPanel` when nodes are selected in StagePlan
- When StagePlan is disabled: existing behavior unchanged

### Step 4: Clean up main `PropertiesPanel`

**File:** `src/components/inspector/PropertiesPanel.tsx`

Remove the StagePlan-specific controls (lines 191-235 — shape dropdown and rotation slider). The main panel should only show main diagram properties.

Keep the `isStagePlanEnabled` import — it's still needed if there are other conditional behaviors.
