# Plan to change new node creation location to the center of the view

## Problem
New nodes are currently created at a fixed position `{ x: 100, y: 100 }` when the user presses Space or Enter in the diagram canvas.

## Goal
Change the creation location to the center of the current visible area (viewport) in the diagram.

## Implementation Steps

### 1. Modify `src/components/diagram/DiagramCanvas.tsx`

- **Import `ReactFlowInstance`**: Add `ReactFlowInstance` to the imports from `@xyflow/react`.
- **Store the ReactFlow instance**:
    - Add a `useRef` to store the `ReactFlowInstance`.
    - Use the `onInit` prop of the `ReactFlow` component to capture the instance and store it in the ref.
- **Update node creation trigger**:
    - In the `useEffect` hook that handles `keydown` events, find the logic for `Space` and `Enter` keys.
    - Instead of hardcoding `setPendingPosition({ x: 100, y: 100 })`, calculate the center of the current viewport:
        - Access the `reactFlowInstanceRef.current`.
        - Get the current viewport using `instance.getViewport()`.
        - Get the container dimensions using `containerRef.current.clientWidth` and `containerRef.current.clientHeight`.
        - Calculate the flow coordinates for the center:
          `x = (containerWidth / 2 - viewport.x) / viewport.zoom`
          `y = (containerHeight / 2 - viewport.y) / viewport.zoom`
        - Call `setPendingPosition({ x, y })`.
    - Fallback to `{ x: 100, y: 100 }` if the instance or container is not available.
    - Call `setIsModalOpen(true)`.

## Verification
- Open the application.
- Pan and zoom the diagram.
- Press Space or Enter.
- Verify that the `NodeCreationModal` appears and that the new node is created at the center of the current view.
