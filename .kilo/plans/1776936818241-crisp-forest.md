# Plan: Refactor NodeListModal.tsx

This plan outlines the steps to break down the `NodeListModal.tsx` component into smaller, more manageable modules and extract shared logic/types.

## Goal
Improve maintainability and readability of `NodeListModal.tsx` by decomposing it into smaller components and extracting utility functions and types.

## Steps

### 1. Centralize Utility Functions
- The `cn` function is already defined in `src/lib/utils.ts`.
- Update `NodeListModal.tsx` to import `cn` from `@/lib/utils` (or relative path) instead of having a local implementation.

### 2. Extract Types
- Create `src/components/diagram/node-list-modal-types.ts`.
- Move the following types to this file:
    - `GroupByMode`
    - `GroupedNode`
    - `NodeListModalProps`

### 3. Extract Logic (Grouping Helpers)
- Create `src/components/diagram/node-list-utils.ts`.
- Move the following functions to this file:
    - `groupByName`
    - `groupByLocation`
    - `groupByType`

### 4. Decompose UI into Sub-components
Create a new directory `src/components/diagram/node-list-modal/` and implement the following:

- `NodeListHeader.tsx`: 
    - Renders the "Existing Nodes" title and the close button.
    - Receives `onClose` as a prop.
- `NodeListTabs.tsx`:
    - Renders the "None", "Location", and "Type" selection buttons.
    - Receives `groupBy` and `setGroupBy` as props.
- `NodeListItem.tsx`:
    - Renders an individual `GroupedNode` row.
    - Receives the node data and the current `groupBy` mode.
- `NodeListContent.tsx`:
    - Renders the scrollable list of nodes.
    - Implements the logic for inserting grouping headers (Location/Type).
    - Receives `aggregatedNodes` and `groupBy` as props.

### 5. Refactor Main Component
- Update `src/components/diagram/NodeListModal.tsx` to:
    - Import all new components, types, and utilities.
    - Remove local `cn`, helper functions, and types.
    - Use the new sub-components within the `createPortal` structure.

### 6. Verification
- Ensure that all imports are correctly resolved.
- Verify that the modal functionality (opening, closing, grouping, scrolling) remains identical to the original implementation.
