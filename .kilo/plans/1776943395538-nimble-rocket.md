# Plan: Add checkbox to toggle "Type and Location" details in NodeListModal

This plan implements a checkbox in the `NodeListModal` that allows users to toggle the visibility of "Type" and "Location" details for each item in the list.

## Steps

1. **Update Types**
   - Modify `src/components/diagram/node-list-modal-types.ts` to add `showDetails: boolean` to `NodeListContentProps` and `NodeListItemProps`.

2. **Implement Toggle in Modal**
   - Modify `src/components/diagram/NodeListModal.tsx` to:
     - Add a `showDetails` state using `useState`.
     - Add a checkbox/toggle UI element below the `NodeListTabs` component to control the `showDetails` state.
     - Pass the `showDetails` state to `NodeListContent`.

3. **Pass State through Content**
   - Modify `src/components/diagram/node-list-modal/NodeListContent.tsx` to accept the `showDetails` prop and pass it to each `NodeListItem`.

4. **Conditionally Render Details in Item**
   - Modify `src/components/diagram/node-list-modal/NodeListItem.tsx` to:
     - Use the `showDetails` prop to conditionally render the `Tag` (Type) and `MapPin` (Location) elements.
     - Ensure the existing `groupBy === "none"` logic for Location is preserved within the `showDetails` condition.

## Verification
- Open the `NodeListModal` in the application.
- Verify that "Type" and "Location" details are visible by default (or as they currently are).
- Click the new checkbox to hide the details.
- Click the checkbox again to show the details.
- Check that the toggle works correctly across different grouping modes ("None", "Location", "Type").
