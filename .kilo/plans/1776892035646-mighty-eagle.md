# Plan for Adding "Cable Type" Property to Edges

## Objective
Add a "Cable Type" property to edges, allow managing cable types in settings, and provide a dedicated inspector module for edge properties when edges are selected.

## Implementation Steps

### 1. Update Store (`src/store/useStore.ts`)
- **State Changes**:
  - Add `cableTypes: string[]` to `DiagramState`.
  - Replace `selectedEdgeId: string | null` with `selectedEdgeIds: string[]`.
- **Actions**:
  - Add `setSelectedEdgeIds: (edgeIds: string[]) => void`.
  - Add `addCableType: (type: string) => void`.
  - Add `removeCableType: (type: string) => void`.
  - Add `updateEdgeCableType: (edgeIds: string[], cableType: string) => void`.
  - Update `deleteEdge` to accept `edgeIds: string[]` and delete all specified edges.
- **Persistence**:
  - Update `restoreProjectState` to include `cableTypes`.
- **Initialization**:
  - Update `onConnect` to ensure new edges have `data: { cableType: 'none' }` (or a default).

### 2. Update Settings Modal (`src/components/settings/SettingsModal.tsx`)
- Add a new section for "Cable Type" list.
- Implement form to add new cable types using `addCableType`.
- Implement list of existing cable types with a delete button using `removeCableType`.

### 3. Update Inspector (`src/components/inspector/`)
- **New Component: `EdgePropertiesPanel.tsx`**
  - Displays properties for selected edges.
  - Shows a `<select>` dropdown for "Cable Type" using `cableTypes` from the store.
  - The selected value in the dropdown should be the `cableType` of the first selected edge.
  - Provides a `Plus` button to add a new cable type via `window.prompt`.
  - `handleAddNewCableType` will:
    - Prompt for a new type name.
    - If valid, call `addCableType(newType)` and `updateEdgeCableType(selectedEdgeIds, newType)`.
  - On `<select>` change, call `updateEdgeCableType(selectedEdgeIds, newValue)`.
- **Update `PropertyInspector.tsx`**:
  - Update logic to decide which panel to show based on selection:
    - If `selectedEdgeIds.length > 0` -> Render `<EdgePropertiesPanel />`.
    - Else if `selectedNodeIds.length > 0` -> Render `<PropertiesPanel />`.
    - Else -> Render `<TemplateList />`.

### 4. Update Diagram Canvas (`src/components/diagram/DiagramCanvas.tsx`)
- **Selection Logic**:
  - Update `onSelectionChange` to handle both `nodes` and `edges`, syncing `selectedNodeIds` and `selectedEdgeIds` with the store.
  - Update `onEdgeClick` and `onNodeClick` to be compatible with the new selection model (or rely on `onSelectionChange`).
  - Update `onPaneClick` to clear both `selectedNodeIds` and `selectedEdgeIds`.
  - Update `onNodeDragStart` to clear `selectedEdgeIds`.
- **Keyboard Handler**:
  - Update the "Delete" / "Backspace" key handler to use `selectedEdgeIds` for calling `deleteEdge`.

## Verification Plan
- Verify that cable types can be added/removed in the Settings Modal.
- Verify that selecting an edge shows the new `EdgePropertiesPanel`.
- Verify that changing the Cable Type in the `EdgePropertiesPanel` updates the selected edge(s).
- Verify that adding a new Cable Type from the `EdgePropertiesPanel` works and applies to selected edges.
- Verify that multi-selection of edges works and allows updating all of them at once.
- Verify that deleting edges works with the new `selectedEdgeIds` logic.
