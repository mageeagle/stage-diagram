# Plan: Implement "Hidden" property for nodes with updated export behavior

This plan outlines the steps to implement a "Hidden" property for nodes that, when enabled, prevents the node and its connected edges from appearing in canvas exports. Instead of complex DOM manipulation, we will temporarily remove hidden nodes and edges from the store during the export process.

## Objective
Add a "Hidden" property to nodes, manageable via the `PropertiesPanel` for single or multiple selections. This property should cause the node and all its connected edges to be hidden during canvas export (PNG, JPG, SVG, PDF) by temporarily removing them from the diagram state during the export operation.

## Implementation Steps

### 1. Data Model Update
- **File**: `src/types/diagram.ts`
- Add an optional `hidden?: boolean;` field to the `CustomNodeData` interface.

### 2. State Management Update
- **File**: `src/store/useStore.ts`
- Update `DiagramState` interface to include `updateNodeHidden: (nodeIds: string[], hidden: boolean) => void;`.
- Implement `updateNodeHidden` in the `useStore` implementation. It should update the `hidden` property in `node.data` for all provided `nodeIds`.
- Update `addNode` and `applyTemplate` to initialize `hidden: false` in the node's data.

### 3. UI Update: Properties Panel
- **File**: `src/components/inspector/PropertiesPanel.tsx`
- Add a "Hidden" checkbox to the property controls.
- **For single selection**:
    - Add a checkbox near the "Require Power Plug?" option.
    - The checkbox should reflect the `hidden` status of the selected node.
- **For multi-selection**:
    - Add a checkbox near the "Require Power Plug?" option.
    - The checkbox should reflect whether all selected nodes are hidden.
- Integrate the `updateNodeHidden` action with the checkbox `onChange` handler.

### 4. Export Logic Update (Amended)
- **File**: `src/components/diagram/ExportButton.tsx`
- Import `useStore` from `@/store/useStore`.
- **Modify `runExport`**:
    1. Capture current `nodes` and `edges` from `useStore.getState()`.
    2. Identify `hiddenNodeIds` (where `node.data.hidden === true`).
    3. Create `exportNodes` by filtering out `hiddenNodeIds`.
    4. Create `exportEdges` by filtering out edges connected to `hiddenNodeIds`.
    5. Temporarily update the store using `useStore.setState({ nodes: exportNodes, edges: exportEdges })`.
    6. Wait briefly (e.g., `await new Promise(r => setTimeout(r, 100))`) to allow the UI to re-render and reflect the change in the DOM.
    7. Perform the export (PNG, JPG, SVG) and handle UI element hiding (controls, minimap, etc.) as before.
    8. In the `finally` block, restore the original `nodes` and `edges` using `useStore.setState({ nodes: originalNodes, edges: originalEdges })`.
- **Modify `exportAsPdf`**:
    1. Follow the same pattern as `runExport`: capture state, filter nodes/edges, temporarily `setState`, wait for re-render, perform the PDF export (via SVG generation), and restore state in the `finally` block.

## Verification Plan
- **Manual Testing**:
    - Select a node and toggle "Hidden" in the `PropertiesPanel`.
    - Verify that the node is still visible in the editor.
    - Export the canvas as PNG, JPG, SVG, and PDF.
    - Verify that the hidden node and all its connected edges are absent from the exported files.
    - Select multiple nodes (some hidden, some not) and toggle "Hidden". Verify bulk update works.
    - Verify that edges connected to hidden nodes are indeed hidden in the export.
- **Code Quality**:
    - Ensure no breaking changes to existing node/edge properties.
    - Ensure all new code follows existing patterns and styles.
    - Verify that `setState` is used correctly to avoid unnecessary history entries (it shouldn't trigger `recordHistory` if used directly).
