# Plan for implementing Node List Modal

## Objective
Create a popup modal that displays a list of all existing nodes in the diagram, with options to group them by their `Location` or `Type` property.

## Implementation Steps

### 1. Update `src/store/useStore.ts`
- Add `isNodeListModalOpen: boolean` to the `DiagramState` interface.
- Add `setIsNodeListModalOpen: (isOpen: boolean) => void` to the `DiagramState` interface.
- Implement `isNodeListModalOpen` and `setIsNodeListModalOpen` in the `useStore` implementation.

### 2. Create `src/components/diagram/NodeListModal.tsx`
- Create a new component `NodeListModal`.
- Use `createPortal` to render the modal at the end of `document.body`.
- Use `useStore` to access the `nodes` array.
- Implement local state for the grouping mode: `'none' | 'location' | 'type'`.
- Implement the UI for the modal:
    - A header with a title and a close button.
    - A control section to select the grouping mode (e.g., buttons or a toggle).
    - A scrollable list area that renders nodes based on the selected grouping.
    - If grouped by `location`, group nodes by `node.data.location`.
    - If grouped by `type`, group nodes by `node.data.type`.
    - If no grouping, just list the nodes.
- Use Tailwind CSS for styling, following the existing modal pattern (like `NodeCreationModal.tsx`).

### 3. Update `app/page.tsx`
- Import `List` icon from `lucide-react`.
- Import `NodeListModal` component.
- Destructure `isNodeListModalOpen` and `setIsNodeListModalOpen` from `useStore`.
- Add a new button to the existing button group in the top-right corner (lines 38-58).
    - The button will use the same styling as the other buttons in that group.
    - The button will trigger `setIsNodeListModalOpen(true)`.
- Add the `NodeListModal` component to the `main` element.

## Verification
- Verify that the new button appears in the correct location with the correct style.
- Verify that clicking the button opens the `NodeListModal`.
- Verify that the modal correctly displays the nodes.
- Verify that switching between grouping modes (None, Location, Type) correctly updates the list.
- Verify that the modal can be closed.
