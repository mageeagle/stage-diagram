# Plan: Group-by-Location Selection and Editing

## Current Architecture

### Grouping System
- Nodes with the same `location` property are automatically grouped when `locationGroupsEnabled` is true
- A "group node" (type: "group") is displayed representing the group
- Individual nodes in a group use `parentId` and `absolute` positioning
- Group node IDs follow pattern: `group-{locationName}`

### Properties Panel Behavior
- **Single Selection**: Shows all properties (Label, Type, Location, Power, Inputs, Outputs)
- **Multi Selection**: Only shows Type, Location, Power (Label hidden, Inputs/Outputs hidden)
- The panel uses `primaryNode` (first selected node) for reading/editing properties
- Changes apply to all selected nodes sharing common property values

## Proposed Implementation

### Phase 1: Group Node Click Handler

**Current**: `onNodeClick` (line 82-88 in DiagramCanvas.tsx) always selects single node
```typescript
const onNodeClick = useCallback(
  (_: React.MouseEvent, node: Node) => {
    setSelectedNodeIds([node.id]);
    setSelectedEdgeIds([]);
  },
  ...
);
```

**Change**: Detect if clicked node is a group node
- If group node: select all member nodes and their connected edges (the group node itself is NOT selected)
- If regular node: current behavior (select single node)

### Phase 2: Group Selection Logic

#### Option A: Flatten Groups on Selection (Recommended)
- When clicking a group node, select all member nodes
- Select all edges connected to these member nodes
- Properties panel will read from first member node (excluding group nodes)
- The group node is not part of the selection (no visual feedback)

**Advantages**:
- Consistent with multi-select UX (hides group node in panel because it's not selected)
- Allows editing individual nodes even when group members are selected
- Group node remains in display but has no visual selection feedback

**Implementation**:
```typescript
const onNodeClick = useCallback(
  (_: React.MouseEvent, node: Node) => {
    if (node.id.startsWith('group-')) {
      const location = node.data.label;
      const memberNodes = nodes.filter(
        n => n.data.location === location && !n.id.startsWith('group-')
      );
      const memberIds = memberNodes.map(n => n.id);
      
      // Select only member nodes
      setSelectedNodeIds(memberIds);

      // Select edges connected to these member nodes
      const connectedEdges = edges.filter(e => 
        memberIds.includes(e.source) || memberIds.includes(e.target)
      );
      setSelectedEdgeIds(connectedEdges.map(e => e.id));
    } else {
      setSelectedNodeIds([node.id]);
      setSelectedEdgeIds([]);
    }
  },
  [nodes, edges, setSelectedNodeIds, setSelectedEdgeIds]
);
```

### Phase 3: Properties Panel Updates

**Current Multi-Select Behavior**:
- Hides Label, Inputs, Outputs (line 32-34 in PropertiesPanel.tsx)
- Shows only Type, Location, Power, action buttons

**Group Editing Specifics**:
- When group members are selected, it behaves as a multi-select
- Location dropdown should probably disable "None" if group is active
- When editing Location of a group, must ensure ALL members get same location

### Phase 4: Edge Selection

- Clicking a group node selects all edges connected to its member nodes.
- Edges can still be selected independently by clicking them directly.

### Phase 5: Keyboard Navigation

Current behavior:
- Click on node → select node
- Space key → open node creation modal

Group-specific considerations:
- Click group member → should this expand group or select just member?
- **Recommendation**: Click group member selects just member (preserves current multi-select)
- Click group header → selects entire group members and their edges as described in Phase 1

### Phase 6: Visual Feedback

**Group Selection State**:
- Group node does **NOT** get `selected: true` (no visual feedback for the group node itself)
- Individual nodes in selected group will show their standard selection indicator
