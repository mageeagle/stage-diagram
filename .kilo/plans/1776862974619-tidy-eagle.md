# Plan: Implement Group Selection Functionality

This plan outlines the steps to implement group selection, copy, delete, and property updates for multiple selected nodes, while maintaining differentiated UI behavior for single vs. multiple selection.

## Objectives
- Enable selection of multiple nodes.
- Implement group actions: Copy, Delete, and Property Updates (Type, Location, Power) for all selected nodes.
- Ensure Undo/Redo works for group actions.
- Update UI (Property Inspector, Diagram Canvas) to support multiple selection.
- Implement differentiated UI in `PropertyInspector`:
  - **Single node selected**: Show all properties (Label, Power, Type, Location, Inputs, Outputs).
  - **Multiple nodes selected**: Show ONLY Type, Location, and Power as group controls.

## Implementation Steps

### 1. Update `useStore.ts` to support multiple selected nodes
- Replace `selectedNodeId: string | null` with `selectedNodeIds: string[]`.
- Implement `setSelectedNodeIds: (nodeIds: string[]) => void`.
- Update `copyNode` to `copyNodes(nodeIds: string[])`:
  - Call `recordHistory()`.
  - For each `nodeId` in `nodeIds`, create a new node with a new `nanoid()` and offset position.
  - Add all new nodes to the `nodes` array.
- Update `deleteNode` to `deleteNodes(nodeIds: string[])`:
  - Call `recordHistory()`.
  - Filter out all nodes in `nodeIds` from `nodes`.
  - Filter out all edges connected to any node in `nodeIds` from `edges`.
  - Clear `selectedNodeIds`.
- Update `updateNodeType`, `updateNodeLocation`, and `updateNodePower` to accept `nodeIds: string[]`:
  - Call `recordHistory()`.
  - For each `nodeId` in `nodeIds`, update the corresponding property in its `data`.
- (Optional/Consistency) Update `updateNodeLabel`, `addInput`, `removeInput`, `updateInputName`, `addOutput`, `removeOutput`, `updateOutputName` to support `nodeIds: string[]`.

### 2. Update `DiagramCanvas.tsx` to synchronize selection
- Add `onSelectionChange` to the `ReactFlow` component to update `selectedNodeIds` in the store.
- Update the `handleKeyDown` event listener:
  - Use `selectedNodeIds` instead of `selectedNodeId` for `Delete`/`Backspace` and `C` keys.

### 3. Update `PropertyInspector.tsx` with conditional UI
- Use `selectedNodeIds` instead of `selectedNodeId`.
- **If `selectedNodeIds.length === 1`**:
  - Use the existing logic to show all properties (Label, Power, Type, Location, Inputs, Outputs) for the single selected node.
- **If `selectedNodeIds.length > 1`**:
  - Display ONLY Type, Location, and Power.
  - The displayed values for these properties will be taken from the first node in `selectedNodeIds`.
  - When Type, Location, or Power is changed, call the multi-node update functions in `useStore` using the full `selectedNodeIds` array.
  - Show the "Copy Node" and "Delete Node" buttons at the bottom.
- This ensures that for a single selected node, it behaves exactly as before. For multiple selected nodes, it provides a streamlined group-edit interface.

### 4. Verification
- Select one node: Verify all properties are visible and editable (matches current behavior).
- Select multiple nodes (using drag selection):
  - Verify `selectedNodeIds` in store is updated.
  - Verify `Delete` key deletes all selected nodes.
  - Verify `C` key copies all selected nodes.
  - Verify `Undo` / `Redo` works for these actions.
  - Verify changing Type, Location, or Power in the `PropertyInspector` updates all selected nodes.
  - Verify that Label, Inputs, and Outputs are NOT displayed when multiple nodes are selected.
