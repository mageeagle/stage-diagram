# Plan: Implement "Stage Plan Edit" Mode

## Overview
Add a toggleable "Stage Plan Edit" mode that creates a separate ReactFlow canvas with nodes initialized with only names (no edges, inputs, or outputs). This allows users to modify node properties without affecting the original canvas.

## Files to Create/Modify

### 1. Create New Store: `src/store/useStagePlanStore.ts`
**Purpose:** Dedicated state management for the stage plan edit canvas.

**Responsibilities:**
- Separate `nodes` and `edges` arrays (empty initially)
- Own `undoStack` and `redoStack` for history management
- Own state for modals, settings, etc.
- All actions mirroring the original store but isolated

**Key Differences from `useStore`:**
- No edges ever created or displayed
- No inputs/outputs on nodes
- No grouping functionality (locationGroupsEnabled not needed)
- Export functionality still supported

### 2. New Component: `src/components/diagram/StagePlanCanvas.tsx`
**Purpose:** Separate component for stage plan edit canvas.

**Changes:**
- Create new dedicated component file
- Render ReactFlow from `useStagePlanStore`
- Clone main canvas nodes when toggling ON
- No edges rendered
- Nodes initialized with only labels (name property)
- Supports undo/redo, export
- Group by location disabled for now

### 3. Modify `src/components/toolbar/Toolbar.tsx`
**Purpose:** Add toggle UI controls.

**Changes:**
- Add button to toggle stage plan edit mode
- Button style matches existing Toolbar buttons
- Add indicator showing current mode status ("Stage Plan Edit" label)
- Export button only exports stage plan canvas when mode is ON

### 4. Modify `app/page.tsx`
**Purpose:** Conditionally render stage plan canvas.

**Changes:**
- Import `useStagePlanStore` and `StagePlanCanvas`
- Control visibility via prop or conditional rendering
- Export main canvas state when toggling to stage plan
- Export stage plan canvas when stage plan mode is active


### 5. Modify `src/components/diagram/ExportButton.tsx`
**Purpose:** Support exporting with canvas-aware logic.

**Changes:**
- New prop `exportFromStore: 'store' | 'stagePlan'`
- When `'stagePlan'`: use `useStagePlanStore` for nodes/edges
- When `'store'` (default): use `useStore` for main canvas
- Export logic same for both, just different data source
- Optional `isStagePlanMode: boolean` prop to conditionally switch store

## Implementation Details

### Stage Plan Canvas Node Data
```typescript
interface StagePlanNodeData {
  label: string;           // Only this property is guaranteed
  type?: string;          // Optional type for CustomNode
  hidden?: boolean;
  power?: boolean;
  // No inputs/outputs
  // No edges
}
```

### Initial State for Stage Plan Store
```typescript
const stagePlanNodes: Node<StagePlanNodeData>[] = [];
const stagePlanEdges: Edge[] = [];
// Always empty initially, clones from main canvas on toggle ON
```

### Node Cloning Logic
```typescript
// When stage plan mode toggles ON, clone main canvas nodes
const clonedNodes = mainCanvasNodes.map((node) => ({
  id: nanoid(),                          // Generate new ID for stage plan
  type: node.type,
  position: node.position,
  data: {
    label: node.data.label,              // Preserve label
    type: node.data.type,                // Preserve type property
    power: node.data.power,
    hidden: false,                       // Reset hidden to false
    hideFromList: false,
    // No inputs/outputs transferred
    // No location transferred (simpler stage plan)
  },
}));
```

**Note:** Only `label` and `type` are preserved from original nodes. Location is not transferred to keep stage plan simpler and focused on node shape/property editing.

### Toggle Logic
```typescript
// When stage plan mode toggles ON:
- Create clone function that reads from useStore
- Clone selected nodes OR all nodes to stage plan canvas
- ExportStagePlan(canvasState) called before switching
- StagePlanCanvas rendered in app/layout.tsx or controlled by parent

// When stage plan mode toggles OFF:
- Unmount StagePlanCanvas component
- Main canvas restored automatically from useStore state
- No modal state transfer needed
```

### Export for Stage Plan
```typescript
interface StagePlanProjectState {

```

## Edge Cases

1. **Existing edges connecting to nodes**
   - Stage plan canvas never has edges
   - When toggling ON, don't create edges - leave nodes isolated

2. **Group by Location in Stage Plan Mode**
   - Location groups functionality still needed even without edges
   - Groups help organize nodes visually
   - Same logic as main canvas but without cables

3. **Concurrent Modals**
   - Stage plan mode modal state separate from main canvas
   - Node creation uses stage plan store
   - Settings can affect both or be independent

4. **Switching Between Modes**
   - Preserve modal state when toggling
   - Don't discard unsaved work
   - Consider: save to JSON file before switching?
