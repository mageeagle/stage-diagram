# Plan: Implement Edit Template Modal

This plan outlines the steps to add an "Edit" functionality for templates in the node-diagram application. Currently, templates can be created and deleted, but not edited.

## Goals
- Allow users to edit existing templates via a modal.
- Add an "Edit" button to each template item in the Property Inspector.
- Ensure template identification is robust by using unique IDs.

## Steps

### 1. Update Types
- **File**: `src/types/diagram.ts`
- **Action**: Add `id: string` to the `NodeTemplate` interface.

### 2. Update Store
- **File**: `src/store/useStore.ts`
- **Action**:
    - Update `DiagramState` interface to include `updateTemplate: (template: NodeTemplate) => void`.
    - Implement `updateTemplate(template: NodeTemplate)` which updates the template in the state by matching its `id`.
    - Refactor `deleteTemplate(templateId: string)` to use `id` instead of `name` for deletion.

### 3. Create Template Edit Modal
- **File**: `src/components/diagram/TemplateEditModal.tsx` (New File)
- **Action**:
    - Create a modal component that uses `createPortal` to render at the document body.
    - The modal will accept `isOpen`, `onClose`, `onSave`, and `template` as props.
    - The form inside the modal will allow editing of:
        - `name`
        - `nodeType`
        - `type` (the data type)
        - `inputs` (array of `NodeInput`)
        - `outputs` (array of `NodeOutput`)
    - Implement logic to add, remove, and rename inputs and outputs within the modal.
    - On "Save", call the `onSave` callback with the updated template.

### 4. Update Property Inspector
- **File**: `src/components/inspector/PropertyInspector.tsx`
- **Action**:
    - Import `TemplateEditModal` and `Pencil` icon from `lucide-react`.
    - Import `nanoid` from `nanoid`.
    - Update `handleSaveAsTemplate` to assign a new `id: nanoid()` to every newly created template.
    - Add local state: `editingTemplate: NodeTemplate | null` and `isTemplateEditModalOpen: boolean`.
    - In the template list mapping:
        - Add an "Edit" button (Pencil icon) next to the "Delete" button.
        - Clicking "Edit" sets `editingTemplate` and sets `isTemplateEditModalOpen` to true.
    - Add the `TemplateEditModal` component to the JSX, wired to the local state and `updateTemplate` action.
    - Update the "Delete" button to pass `template.id` instead of `template.name` to `deleteTemplate`.

## Verification
- Verify that templates can be created with an ID.
- Verify that clicking "Edit" opens the modal with the correct template data.
- Verify that editing the name/type/inputs/outputs and saving updates the template in the list.
- Verify that clicking "Delete" correctly removes the template using its ID.
- Verify that "Cancel" in the modal does not change the template.
