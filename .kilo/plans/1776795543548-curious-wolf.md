# Implementation Plan: Node Templates

## Goal
Implement a "Templates" feature that allows users to save the configuration of a node (Name, Inputs, Outputs, and Type) as a reusable template and add these templates to the canvas via the Property Inspector.

## Proposed Changes

### 1. Data Models (`src/types/diagram.ts`)
- Define a `NodeTemplate` interface:
  ```typescript
  export interface NodeTemplate {
    name: string;
    inputs: NodeInput[];
    outputs: NodeOutput[];
    type: string;       // The 'type' property within node.data
    nodeType: string;   // The 'type' property of the Node itself (React Flow type)
  }
  ```

### 2. State Management (`src/store/useStore.ts`)
- Update `DiagramState` to include:
  - `templates: NodeTemplate[]`
  - `addTemplate: (template: NodeTemplate) => void`
  - `useTemplate: (template: NodeTemplate, position: { x: number; y: number }) => void`
- Implement `addTemplate`: Adds a new template to the `templates` array.
- Implement `useTemplate`: Creates a new `Node` using the template's properties and adds it to the `nodes` array.

### 3. Property Inspector (`src/components/inspector/PropertyInspector.tsx`)
- **Scenario A: Node Selected**
  - Add a "Save as Template" button in the action area (bottom of the inspector).
  - When clicked, extract properties from the `selectedNode` and call `addTemplate` using the current node's label as the template name.
- **Scenario B: No Node Selected**
  - Replace the "Select a node to edit properties" placeholder with a list of available templates.
  - Each template will be displayed as a clickable item.
  - Clicking a template will call `useTemplate` with a default position of `{x: 0, y: 0}`.
