# Plan: Wire Toolbar Undo/Redo and ExportButton to StagePlan Store

## Goal

When stage plan mode is active (`isStagePlanEnabled` is true), the toolbar undo/redo buttons, keyboard shortcuts, and ExportButton should operate on the `useStagePlanStore` instead of `useStore`.

## Changes

### 1. `src/components/toolbar/Toolbar.tsx` — Conditional undo/redo

**Problem:** Undo/redo always come from `useStore` (lines 37-38), regardless of mode. The toolbar already imports `useStagePlanStore` and tracks `isStagePlanEnabled` (line 39).

**Changes:**
- Add `undo` and `redo` selectors from `useStagePlanStore`
- Create two wrapper functions `handleUndo` and `handleRedo` that check `isStagePlanEnabled` and call the correct store's undo/redo
- Update the `buttons` array to use `handleUndo` / `handleRedo` instead of the raw `undo` / `redo` from `useStore`

```tsx
// Add these selectors:
const spUndo = useStagePlanStore((state) => state.undo);
const spRedo = useStagePlanStore((state) => state.redo);

// Create conditional handlers:
const handleUndo = () => isStagePlanEnabled ? spUndo() : undo();
const handleRedo = () => isStagePlanEnabled ? spRedo() : redo();

// Use in buttons array:
{ key: "undo", title: "Undo", onClick: handleUndo },
{ key: "redo", title: "Redo", onClick: handleRedo },
```

### 2. `src/components/diagram/StagePlanCanvas.tsx` — Keyboard shortcuts

**Problem:** `handleKeyDown` (lines 103-122) only handles Delete/Backspace. The `undo` and `redo` functions are already imported from `useStagePlanStore` (lines 53-54) but never used in keyboard handling.

**Changes:**
- Add Ctrl+Z (undo) handler
- Add Ctrl+Y and Ctrl+Shift+Z (redo) handlers
- Follow the same pattern as DiagramCanvas (lines 229-243)

```tsx
if ((event.ctrlKey || event.metaKey) && (event.key === "z" || event.key === "Z")) {
  undo();
}

if ((event.ctrlKey || event.metaKey) && (event.key === "y" || event.key === "Y" || (event.shiftKey && event.key === "Z"))) {
  redo();
}
```

Add `undo` and `redo` to the `useCallback` dependency array for `handleKeyDown`.

### 3. `src/components/diagram/StagePlanCanvas.tsx` — ExportButton mode

**Problem:** `ExportButton` is rendered at line 200 without `isStagePlanMode` prop, so it defaults to reading from `useStore` instead of `useStagePlanStore`.

**Change:** Pass `isStagePlanMode={true}` to ExportButton:

```tsx
<ExportButton targetRef={containerRef} isStagePlanMode={true} />
```
