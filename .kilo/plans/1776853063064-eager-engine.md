# Plan: Add "Power" property to nodes

## Goal
Add a boolean "Power" property to each node, making it editable via a checkbox in the Property Inspector with the label "Require Power Plug?".

## Implementation Steps

### 1. Update Types
- Modify `src/types/diagram.ts`:
    - Add `power: boolean` to `CustomNodeData`.
    - Add `power: boolean` to `NodeTemplate` to ensure templates can store and restore the power state.

### 2. Update Store
- Repair and Update `src/store/useStore.ts`:
    - Remove all duplicate and malformed code blocks.
    - Ensure the `DiagramState` interface includes `updateNodePower: (nodeId: string, power: boolean) => void`.
    - Implement `updateNodePower` in the `useStore` implementation.
    - Update `addNode` signature to include an optional `power: boolean` parameter and use it to initialize `newNode.data.power`.
    - Update `applyTemplate` to include the `power` property from the template in the new node's data.
    - Verify `copyNode` correctly preserves the `power` property via object spreading.

### 3. Update Property Inspector
- Modify `src/components/inspector/PropertyInspector.tsx`:
    - Destructure `updateNodePower` from `useStore`.
    - Add a new UI section for the "Power" property, featuring a checkbox with the label "Require Power Plug?".

## Verification
- Create a node and check if it has the "Power" property (implicitly via Inspector).
- Toggle the "Power" checkbox in the Inspector and verify it updates the node's state.
- Create a template with a specific "Power" setting and verify that applying the template restores that setting.
- Copy a node and verify the copied node has the same "Power" state.
